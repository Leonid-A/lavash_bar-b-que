# UI Guidelines

## Purpose

This document defines the visual design principles for the Restaurant Digital Menu.

Every UI implementation should follow these guidelines to maintain consistency.

---

# Design Philosophy

The interface should feel:

- Elegant
- Premium
- Clean
- Modern
- Minimal

Avoid visual clutter.

The menu should remain easy to read in different lighting conditions.

---

# Mobile First

The application is primarily used on mobile devices.

Design for mobile first.

Desktop should enhance the experience without changing the overall layout.

---

# Layout

Maximum content width:

1200px

Content should be centered.

Use generous whitespace.

Avoid cramped layouts.

---

# Typography

Create a clear hierarchy.

Restaurant Name

- Largest text

Category Titles

- Large
- Bold

Menu Item Name

- Medium
- Semibold

Description

- Smaller
- Neutral color

Price

- Prominent
- Easy to locate

Labels

- Small badges

---

# Colors

Use a neutral color palette.

Prefer:

- White backgrounds
- Dark text
- Soft gray borders
- One primary accent color

Avoid excessive colors.

Labels may use subtle semantic colors.

---

# Spacing

Maintain consistent spacing throughout.

Prefer 4px spacing increments.

Examples:

- Between sections
- Between cards
- Between text elements

Spacing should feel balanced.

---

# Cards

Menu cards should have:

- Rounded corners
- Soft shadow
- Comfortable padding
- Consistent spacing

Cards should never feel crowded.

---

# Images

Menu images are optional.

When present:

- Consistent aspect ratio
- Object-fit cover
- Rounded corners

Missing images should not negatively affect layout.

---

# Buttons

Buttons should have:

- Rounded corners
- Visible hover state
- Visible focus state
- Smooth transitions

Primary actions should stand out.

---

# Search

Search input should:

- Be easy to tap
- Have generous padding
- Include a search icon
- Display a clear placeholder

---

# Category Navigation

Sticky navigation.

Requirements:

- Horizontal scrolling on mobile
- Active category highlighted
- Smooth transitions
- Easy thumb interaction

---

# Animations

Animations should be subtle.

Allowed:

- Fade
- Color transitions
- Hover transitions
- Smooth scrolling

Avoid:

- Large movements
- Bouncing
- Flashing effects

---

# Footer

The footer should remain simple.

Display only essential information.

Avoid visual clutter.

---

# Accessibility

Maintain sufficient contrast.

Touch targets should be comfortable.

Keyboard navigation should work.

Focus states must remain visible.

---

# Responsiveness

Support:

- Mobile
- Tablet
- Desktop

No horizontal overflow.

Layouts should adapt naturally.

---

# Consistency

Every new component should match the existing design language.

Do not introduce different styles for similar components.
